[[编程常用关键字]]
1. final 关键字（最后的，最终的）
可以用来修饰类、属性、方法、局部变量
	在某些情况下，可以用final修饰：
		1)  当不希望类被继承时
		2) 当不希望父类的某个方法被子类覆盖/重写（override）时
		3) 当不希望类的某个属性被修改时
		4) 当不希望某个局部变量被修改时
		5) 一般来说，如果一个类是final类了，没必要将方法构造成final方法
		6) final不能修饰构造方法
		7) final和static往往搭配使用，效率更高，不会导致类加载，底层编译器做了优化
		8) 包装类都是final类，不能被继承
	*注意事项：*
	- final定义的属性称为常量，命名格式：==X_X_X==
	- final修饰的属性定义时必须赋初值，且以后不能更改
		- 赋值位置：
			- 1. 定义时
			- 2. 在构造器中
			- 3. 在代码块中
	- 如果final修饰的是静态属性，初始化位置不能在构造器中
	- final类不能被继承，但可以实例化对象
	- 类不是final类，但是含有final方法，则该方法不能被继承，可以被重写
2. final和static比较
	
|     | 修饰符          | 内存份数     | 值是否能修改 | 调用方法         |
| --- | ------------ | -------- | ------ | ------------ |
|     | final        | 全类共用 1 份 | 不可以    | 类名.变量名（A.a）  |
|     | static       | 每个对象 1 份 | 可以     | 对象名.变量名（A.a） |
|     | final static | 全局仅 1 份  | 不可以    | 类名.常量名（A.a）  |
