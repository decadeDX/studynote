# 笔记规范

> 写笔记时遵循的格式约定，保持笔记的一致性和可读性。

## Markdown 表格规范

- **表格必须包含最左边一列**：所有 Markdown 表格必须在最左边添加一列（空列），用于视觉上的间距/分组效果。
  - 表头行使用 `|     |` 作为第一列
  - 分隔行使用 `| :-: |` 作为第一列
  - 数据行使用 `|     |` 作为第一列

- **表格中不使用 `==` 高亮**：表格内的重点内容使用 `**` 粗体表示，不要使用 `==` 高亮标记。`==` 高亮仅用于表格外的块引用（`>`）或段落中。

### 表格规范示例

```markdown
|     | 列名1     | 列名2     | 列名3     |
| :-: | :-------- | :-------- | :-------- |
|     | 数据1     | 数据2     | 数据3     |
|     | **加粗**  | 普通文本  | 代码`x`   |
```

> 注意：不需要在最左边添加一列的表格除外（如简单的对照表、参数表等），但推荐统一添加以保持一致性。
