# git

## 推送被拒（push rejected）且本地有敏感配置文件

### 问题描述

执行 `git push` 时遇到：

```
 ! [rejected]        master -> master (fetch first)
error: failed to push some refs to '...'
```

同时本地有敏感配置文件（如 `application.yml`）**不想上传**。

### 解决步骤

1. **确认工作区状态**
   ```bash
   git status --short
   ```

2. **检查敏感配置是否被 Git 跟踪**
   ```bash
   git ls-files *application*
   ```

3. **验证 .gitignore 规则是否生效**
   ```bash
   git check-ignore -v yuncang-back/src/main/resources/application.yml
   ```
   > 若 `git ls-files` 未列出、`git check-ignore` 匹配到规则，则文件安全。

4. **同步远程**
   ```bash
   git fetch origin
   git rev-list --left-right --count master...origin/master
   ```

5. **暂存未提交改动（如有）**
   ```bash
   git stash push -u -m "before-rebase"
   ```

6. **变基到远程最新**
   ```bash
   git rebase origin/master
   ```

7. **恢复暂存**
   ```bash
   git stash pop
   ```

8. **再次确认敏感配置未进 Git**
   ```bash
   git ls-tree -r --name-only HEAD | findstr application
   git check-ignore -v yuncang-back/src/main/resources/application.yml
   ```

9. **推送**
   ```bash
   git push origin master
   ```

### 补充：如果配置文件曾被 Git 跟踪过

```bash
git rm --cached yuncang-back/src/main/resources/application.yml
```

并在 `.gitignore` 添加：

```
/src/main/resources/application.yml
```

> ⚠️ **避免使用 `git push --force`**，除非确定要覆盖远程历史。`fetch + rebase + push` 通常就够了。
