## 题目大意

给定一个长度为 `n` 的整数**山脉**数组 `arr`，其中的值递增到一个**峰值元素**然后递减。
返回峰值元素的下标。
你必须设计并实现时间复杂度为 `O(log(n))` 的解决方案。

## 解题思路

### 1. 暴力思路（最初想法）

逻辑：线性的双指针遍历

```text
while 只要左右指针没相遇：

1. 先走左指针判断，满足上升就右移，不满足直接返回 left；
2. 左指针处理完，再走右指针判断，满足下降就左移，不满足直接返回 right；
3. 直到 left == right，循环结束，返回 left（峰顶）。
```
时间复杂度：O(n)
缺点：超时/空间过大，无法通过大数据用例

### 2. 最优解法（最终AC方案）

核心原理（算法关键点）：二分法

步骤拆解：
1. 初始化左右指针 `left = 0, right = arr.length - 2`（山脉数组的峰值不可能是最后一个元素）
2. 当 `left + 1 < right` 时循环：
   - 计算中点 `mid = left + (right - left) / 2`
   - 若 `arr[mid] > arr[mid + 1]`，说明在下降段，峰值在左侧 → `right = mid`
   - 否则说明在上升段，峰值在右侧 → `left = mid`
3. 循环结束，返回 `right`（峰值索引）

### 暴力解法

```java
public int peakIndexInMountainArray(int[] arr) {
    int left = 0;
    int right = arr.length - 1;
    while (left < right) {
        if (arr[left] < arr[left + 1])
            left++;
        else
            return left;
        if (arr[right] < arr[right - 1])
            right--;
        else
            return right;
    }
    return left;
}
```

### 二分法

```java
public int peakIndexInMountainArray(int[] arr) {
    int left = 0;
    int right = arr.length - 2;
    while (left + 1 < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] > arr[mid + 1]) {
            right = mid;
        } else {
            left = mid;
        }
    }
    return right;
}
```

def peakIndexInMountainArray(self, arr: List[int]) -> int:
    left, right = 0, len(arr) - 1
    while left < right:
        mid = (left + right) // 2
        if arr[mid] < arr[mid + 1]:
            left = mid + 1
        else:
            right = mid
    return left
