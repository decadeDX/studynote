## 标识符

- 标识符就是名字：函数名，变量名，类名，对象名等

- 标识符命名规则：

  - 不能使用C++关键字作为标识符。

  - 大小写敏感。

  - 只能由字母、数字、下划线组成，且不能以数字开头。尽量不要使用下划线开头。

  - 标识符要见名知意。

- 标识符命名规范：

  - 普通变量（函数内的变量）命名：类型缩写+单词（首字母大写），例如：nAge，sName。

  - 类的成员变量命名：m_类型缩写+单词（首字母大写），例如：m_nAge，m_sName。

  - 全局变量命名：g_类型缩写+单词（首字母大写），例如：g_nAge，g_sName。

  - 常量命名：全大写，用下划线分割单词。例如：MAX_LENGTH。

  - 函数名：动词+名词组成，第一个单词全部小写，其他单词首字母大写。例如：getAge()，setName()。

  - 类名：多个单词组成，每个单词首字母大写。例如：StudentInfo。

## 变量

- 变量是命名的其值可以改变的量，变量作用是保存数据。

- 一个变量有三个要素：类型、变量名和值。

- 变量的三种初始化方式：

  - 传统初始化方式（`=`）

  - 对象方式初始化方式（`（）`）

  - C++11标准，统一初始化方式（`{}`）

- 定义变量的注意事项

  - 变量使用前必须定义，确定变量名、类型及初值；

  - 变量类型可以是C++支持的所有类型，包括基本类型和自定义类型；

  - 同一个作用域不能定义相同名称的变量。

  - 变量定义时系统会根据类型分配相应大小的内存空间，如果时自定义类型，则会执行相应的构造函数。

  - 相同类型的多个变量可以在一个语句中定义，变量名用逗号分隔。

## 常量

- 在程序运行过程中，其值不能改变的量称为常量。C++中常量分为直接常量和符号常量（命名常量）。

- 直接常量：直接使用数字或文字表示的值。

- 符号常量：使用标识符表示数值或文字，符号常量只能在定义时指定其值，一旦初始化后不能用复制语句修改。符号常量有两种定义方式：

  - 使用宏定义： #define 宏名 值

- 地址编码的最基本单元是字节，每个字节由8位二进制组成。基本内存单元的大小就是1字节，每个单元都有唯一的地址，这个存储单元可以存放指定的数据。

# 八、常量指针与指针常量

## 常量指针

- 指向常量的指针，指针指向的内存地址的内容不可修改。

- 只有常量指针才能指向常量。    

```c
示例：
//可以改变指向，但不能改变值
int a = 100, b = 200;
const int* ptr = &a;//常量指针
ptr = &b;
printf("%d", *ptr);
```

## 指针常量：

- ​	指向的内存中的值可以改变；但是指向的内存地址不能改变。

```c
示例：
//可以改变值，但不能改变指向
int* const p1 = &a;//指针常量
*p1 = 300;
printf("%d\t%d", *p1, a);
```

## 区分

常量指针本质是指针，const修饰的是int*，说明整体的值不可改变，但是指针指向（即存入的地址）可以改变；

指针常量本质是常量，const修饰的是指针本身，说明指针指向（即存入的地址）不可改变，而地址对应的值可以改变



## 空指针和野指针

###    空指针

        NULL代表空指针，就是值为0的指针。有些时候指针声明时候并不知道应该保存哪个地址，这个时候就可以将指针置为NULL。

###   野指针

        指向了不能合法访问的空间的指针就是野指针。

# 九、动态内存分配

- C++内存区

  - 栈区
    - 由编译器自动分配释放，存放为运行程序而分配的局部变量、函数参数、返回值。程序结束后由系统释放。

  - 堆区
    - 由程序员分配释放，若程序员不释放则会产会生内存泄漏。

  - 全局区
    - 存放全局 变量，静态数据，常量，程序结束后由系统释放。

  - 文字常量区
    - 常量字符串就放在这里，程序结束后由系统释放。

  - 程序代码区
    - 存放函数体的二进制代码，由操作系统管理。

- C++内存分配方式有三种

  - 从栈上分配，系统自动分配。

  - 从全局区分配，系统自动分配。

  - 从堆上分配，称为动态分配，需要手动管理。

- 管理内存

  - new、delete分配、释放内存空间

    - 单个存储空间

      ![image](..\picture\c++\基本知识\单空间动态分配.png)

      

      - 申请单个存储空间：new 类型名(初值)

      - 释放单个存储空间:：delete 地址

    - 多个连续存储空间

      ![image](..\picture\c++\基本知识\连续空间动态分配.png)

      - 申请多个连续存储空间: new 类型名{长度}，长度为整型变量或表达式

      - 释放多个连续存储空间：delete[] 连续内存空间首地址。

      - 释放内存后，一般会将指针置空，即 指针变量名=NULL;

  - malloc和free也可以进行动态内存分配，但是C++推荐使用new和delete，这种方式更加安全简洁。

  - new与malloc区别

    - 使用上：malloc要显式填入申请内存的大小，new不需要。

    - 返回值：malloc返回void* 类型，需要强制转换成需要的类型，new会返回分配时指定的类型，无需强制转换。

    - 分配失败时：malloc会返回NULL，new会抛出异常。

# 十一、函数



## 什么是函数

- 在程序设计时，如果一段代码重复进行某个操作或完成特定的功能，则将这样的代码组织为函数。

- 函数定义

  - 函数定义由函数类型、函数名称和参数组成，称为函数头。由大括号括起来的语句为函数体。  

  - 函数参数:

    - 形参（形式参数）：定义函数时括号中的参数

    - 实参（实际参数）：调用函数时括号中的参数

    - 函数调用时，系统为形参分配内存单元并将实参的值传递到形参的单元中。

    - 根据函数功能的需要，函数参数可以选择数值，指针或引用三种形式之一。

      - 数值做参数时，实参与形参在内存中占用不同的存储单元。

      - 指针、引用作为函数参数时，传递的是变量的地址。

    - 数组作为函数的参数和返回值

      - 数组名作为参数实际上传递的时数组的首地址。

      - C++中函数一般不能返回数组，但是可以通过指针实现返回一个数组。



## 变量的生命周期

- 变量的生命周期

  - 每个标识符都有确定的说明位置，决定了它将在什么范围内有效（能被访问）。变量的声明周期也称变量的作用域，即变量的有效i范围。

  - 变量的作用范围

    - 局部变量
      - 函数内部定义的变量是内部变量，也称局部变量。它只在该函数范围内有效。

    - 全局变量
      - 函数外部定义的变量称为外部变量，也称全局变脸。在程序运行过程中都有效。

    - 局部变量和全局变量

      - 在一段程序中，如果出现同名的局部变量和全局变量，优先访问局部变量。

      - 可以使用 ::变量名 访问全局变量

  - 变量的存储类别

    - 静态存储方式
      - 静态存储变量是指在程序运行期间，分配固定的存储空间并且一直存在。

    - 动态存储方式
      - 动态存储变量是在程序执行过程中根据需要动态分配存储空间。

    - static关键字
      - static关键字可以把变量定义为静态存储方式。

# 三、类与对象

### 类的数据成员与函数成员

#### 数据成员

- 类中的数据成员描述该类的对象的属性。器定义方式与一般变量相似，当对数据城的访问要控制权限。

- 数据成员初始化与变量初始化形式有所不同，不能使用()号，可以使用=或{}

- 数据曾元的类型可以是C++支持的任意类型。

- 多个数据成员之间不能重名，一个类作为一个作用域，不允许命名冲突。

#### 类的成员函数

- 成员函数描述类对象的行为，即该对象所能执行的操作，向类外提供服务。

- 一个类中的多个成员函数可以重载；

- 成员函数也可以是内联函数。

- 成员函数的形参也可以带有默认值。

#### 成员访问控制

- 关键字private,protected，public的作用方位是从该关键字开始直到遇到下一个关键字结束。

- private私有访问权限
  - 私有成员只允许本类的成员函数访问，对类的外部不可见。

- protected保护访问权限
  - 能被本类和本类的派生类的成员函数访问，对类的外部不可见。

- plublic共有访问权限
  - 共有成员外部可见，类内部也能访问。

#### 类与对象的关系

- 类与对象是抽象与具体的关系。类是创建对象的样板，由这个样板可以创建多个具有相同属性的行为的对象。

- 类本质上是一种数据类型，对象是某个类的实例，一个类的多个对象分别拥有自己的属性且相互独立。

## 类中的特殊成员

- 类的成员除了普通的数据成员和成员函数，也包括构造函数、析构函数、拷贝构造函数等等，这些函数为支持对象生命周期，从创建开始，调用函数操作对象，然后被对象给释放，结束生命周期。

### 构造函数

#### 概念

- 在定义变量的时候可以为变量提供初始值，同样在创建对象的时候也可以为对象的属性提供初始化，也就是对象的初始化，C++提供了构造函数来完成这个操作。

#### 特点

- 1、构造函数的函数名和类名完全相同

- 2、不能定义构造函数的类型

- 3、构造函数应该声明为公有权限

- 4、构造函数不需要手动调用，会在创建对象的时候自动调用

#### 注意点：

- 1、构造函数可以重载，编译器会根据实际实参决定到底调用哪一个构造函数

- 2、如果类中没有显示声明构造函数，C++编译器会自动生成一个默认的构造函数，没有参数，一旦用户手动声明，默认的构造函数就不再生成了

- 3、无参默认构造函数结构体：`Rect::Rect(){}`

- 4、无论以什么样的方式来定义对象，都会调用构造函数，如果编译器找不到匹配的构造函数，就编译报错。一般为了让对象能实现多种形式的初始化，可以为类定义多个不同参数的构造函数。

### 析构函数

#### 概念

- 和变量一样，函数中定义的局部变量，当程序离开函数的时候，变量就消失，由于类中数据成员可能有指针，对象中就可能出现动态分配的内存，就需要在析构函数中，将动态内存给释放掉，也就是在对象消失之前（析构函数就是在对象消失之前被自动调用）

#### 析构函数特点

- 析构函数的名字要以"~"开头

- 析构函数没有参数，也没有返回值

- 一个类中只有一个析构函数，不能重载

- 当一个对象消失的时候，编译器会自动调用析构函数

- 如果没有显示定义析构函数，编译器会自动生成一个公有的析构函数，函数体为空

#### 主要作用：

在删除一个对象之前被调用，释放该对象成员的内存空间以及一些其他的清理工作（例如：关闭文件、关闭数据库链接）

### 拷贝构造函数

#### 概念

- 创建一个对象可以直接创建出来，也可以由一个已有的同类对象复制而来，即克隆一个对象，后者需要调用拷贝构造函数。

#### 特点：

- 定义方式：`类名 (const 类名& 对象名)`;

- 函数的形参必须为常引用

- 必须通过已有对象创建新对象

#### 拷贝构造函数的形参为什么一定要写成常引用

- `&`（引用）：避免无限递归调用拷贝构造，同时提升效率；
- `const`：保证不修改原对象，兼容 const 对象的拷贝，符合编程规范
- 常引用在接受原对象的时候，拿到的是源对象本身，如果不写引用，会创建一个临时对象会消耗资源，为了避免实参的值被修改，所以要将参数声明为const类型，这样在拷贝的时候既减少了资源的消耗，也阻止了对源对象的修改。

### 赋值函数

用赋值语句把一个对象的赋值给已经存在的同类对象时，将调用赋值函数。

- 一般格式：`类名& operator=(const 类名& 形参){函数体；return *this}`

**`this`就是 “当前对象” 的地址，`*this`就是当前对象本身。**

## 类的三大特性

### 封装性

### 继承性

- **子类不继承父类的构造方法,需要定义自己的构造方法，一般情况下，子类的构造方法的初始化列表中要调用父类构造方法。**

- 析构顺序

  - 执行派生类自己的析构函数

  - 然后按照数据成员声明的相反顺序依次调用各个数据成员的析构函数

  - 按照基类声明的相反顺序依次调用各个基类的析构函数

### 多态性

- 一般指的是一个事物的多种形态，C++语言中多态性分为编译时的静态动态和运行的时的动态多态，后者主要通过继承和虚函数来实现。

- 父类指针指向子类对象是可以的，但是无法直接调用子类的成员函数，必须依赖虚函数才能用父类指针调用子类对象的函数

#### 虚函数定义方式

- virtual 函数类型 函数名(参数表);

- 注意点：

  - 父类有虚函数，子类中的同名函数会自动成为虚函数，子类中的同名函数可以加virtual也可以不加。

  - 虚函数在声明的时候需要加virtual关键字，类外定义的时候是不能写virtual的

- C++运行时的多态主要目的：可以通过基类指针来直接访问所有派生类（包括直接派生和间接派生）的成员函数，对其进行全方位的访问，在不同的继承关系中，调用同一个函数，能够产生不同的行为。

#### 基于虚函数实现运行时的多态需要满足的条件

- 基类中定义了虚函数

- 子类中重写了父类的虚函数

- 用基类指针来调用子类对象并且使用这个虚函数

#### 虚函数实现动态多态的机制

##### 图示

![img](..\picture\c++\基本知识\虚函数实现动态多态的机制.jpg)

- 编译器为每个包含虚函数的类创建一个虚函数表，并且为每一个对象创建一个虚函数指针（虚指针）指向这个类的虚函数表，使用基类指针对虚函数调用的时候，其实通过这个虚指针，在虚函数表中查找相应的虚函数地址，从而完成调用。

#### 虚析构

- 如果基类指针指向派生类的对象，当调用delete释放对象的时候，如果父类的析构函数不是虚函数，那么就无法用基类指针调用子类的析构函数，所以就会导致子类动态申请的空间无法被释放，所以基类需要将析构函数定义为虚函数，就可以将父子类中析构函数都进行调用，空间都可以进行释放。

#### 纯虚函数和抽象类

- 纯虚函数是没有函数体的特殊虚函数，主要的目的是为了给程序提供统一的接口，具体的定义由子类来完成。

- 声明方式：virtual 函数类型 函数名(参数表) = 0;

- 声明纯虚函数之后，不写函数定义，有纯虚函数的类叫做抽象类，通常作为派生类的基类来使用，但是如果派生类继承了抽象类的虚函数，但是没有定义，那么派生类也会自动成为抽象类，除非当前类已经将继承下来的抽象类中纯虚函数全部定义，或者当前派生类自己有新的虚函数，也已被定义，否则就是抽象类。

#### 重载、重写和覆盖的区别

- 重载：通过函数的形参类型或者个数不同，来定义多个同名函数实现一个函数名根据形参不同调用不同的函数，实现不同的结果。

- 重写：如果父类有虚函数，子类将父类的虚函数重新定义，就是函数的重写。

- 覆盖：父类和子类有相同名称的函数，父类的函数不是虚函数，就叫做覆盖。

#### C++11中的关键字

- override：可以检测当前的函数是否是继承下来的虚函数

- final：表示某个虚函数已经是最终版本，子类不能再进行重写

- 图示

- 类的属性不仅可以是基本的数据类型，也可以是类的对象，类的组合就是在一个类中内嵌其他对象作为成员，所以在创建组合类的时候，应该优先创建内嵌对象，就要调用内嵌对象的构造函数，如果构造函数有参数，就必须要传参。

# 一、STL模板库简介

-  STL（Standard Template Libray）标准模板库是C++标准库重要的组成部分，提供了一组通用的模板类和函数，用于实现常用的数据结构和算法。

- STL的涉及基于泛型编程的思想，具有高效，灵活和可复用的特点，其优势在于代码复用，类型安全和高性能，是现代C++编程不可或缺的工具。

- STL主要由以下四个核心组件构成

  - 容器用于存储和组织数据，通常分为以下三类：

    - 序列式容器：按线性顺序存储元素，例如: vector,list,deque,string

    - 关联式容器：基于键自动排序，支持基于键的查找、插入，删除，例如: set,map

    - 容器适配器：不是独立的容器，而是基于其它序列容器封装而成，例如：stack,queue

  - 迭代器
    - 提供统一方式访问容器中的元素。

  - 算法
    - 算法通过迭代器与容器解藕，实现高度通用性。例如：排序，查找，拷贝，反转等。

  - 函数对象
    - 常用于定制算法行为，是重载了operator()的类的对象，可像函数一样调用。例如：比较，排序规则。

# 二、泛型编程

## 什么是泛型编程

- 泛型编程是一种以“通用性”和“抽象类型”为核心的编程方式。

- 泛型编程的核心思想是在设计函数时，不预先限定操作的具体类型，而是让类型成为“参数”，等到使用时再指定实际类型。

- C++中实现泛型编程的具体方式是模板

## 模板概述

- 模板是C++支持泛型编程的核心机制，用于编写与类型无关的通用代码，允许一个函数或类处理多种数据类型。

- 模板的核心价值：提高代码复用性，避免重复编写仅类型不同的函数或类。

- C++提供的模板由函数模板和类模板两种。

## 函数模板

- 传参时分为显示指定和自动推导两种方式

  - 自动类型推导

    - 编译器根据实参T推断，

    - 自动推导要求所有参数能推导出一致的T，否则报错

    - 若无法推导，必须显示指定类型。

  - 显式指定类型
    - 手动指定T的具体类型

  - 普通函数与函数模板的区别

    - 普通函数调用时可以发生自动类型转换(隐式类型转换)。

    - 函数模板调用时，如果利用自动类型推导，不会发生自动类型转换；如果利用显式指定，可以发生自动类型转换。

- 模板的重载

  - 函数模板可以重载，可以定义多个同名但参数列表不同的函数模板。

  - 可以为特定类型提供非模板的普通函数重载，与函数模板共同构成重载集。

  - 当普通函数与函数模板重载时，优先使用普通函数重载。
    - 例如，当需要为特定类定制实现时，使用普通函数为特定类定制实现，当实参为特定类时，优先使用普通函数。

  - 调用规则

    - 若存在多个函数模板候选，优先选择约束更强、适用范围更窄的那个函数模板
      - 例如: T* 比T更特化

    - 若普通函数与模板实例所需的参数转换序列等级相同，则优先选择普通函数。C++将转换分为三个等级（从高到底）：

      - 精确匹配，例如 int ->int, int&->int&，数组名退化为指针等。

      - 提升，例如 ：char -> int, float -> double

      - 转换，例如：int->double, double -> int ,

    - 即使存在更优秀的普通函数重载，也可以通过func<>()显式指定使用函数模板。

## 类模板

- 类模板的作用是定义一个通用类，其成员类型由模板参数决定。

- 特点

  - C++11之前，不支持自动类型推导，必须显式指定类型。这种方式与函数模板不同。

  - 支持默认模板参数。

  - 在实例化时，会用具体参数类型替换模板参数。

- 类模板对象作为函数参数共有三种传入方式

  - 指定具体模板实参（固定类型）

    - 函数形参使用类模板的具体实例化类型，仅接受该特定类型的对象，无法接受其他模板参数组合的对象

    - 类型安全，编译器绑定

    - 当函数逻辑仅针对某个确定类型的模板实例时使用。

  - 参数模板化

    - 将函数定义为模板，其形参类型为类模板的泛化形式，例如Person<T1, T2>

    - 支持模板类泛化参数，且能在函数体内获取并使用模板参数的实际类型。

    - 当需要处理同一类模板的不同实例，且可能依赖内部类型信息时使用。

  - 完全泛化

    - 函数模板使用泛型，不关心传入对象是否来自某个特性的类模板，只要求其具备所需的方法。

    - 使函数拥有最大程度的泛用性，但失去对特定类结构的编译期约束。

- 子类继承模板父类

  - 策略一，在继承时指定父类具体类型，子类可以是普通类也可以是模板类

  - 策略二，父类与子类共享模板参数，父类类型虽子类变化。

# 四、vector动态数组

![vector](..\picture\c++\STL\vector.png)

- vector是可以存储任意类型数据的顺序容器，vector的数据结构和数组非常相似，也成为动态数组。

## vector与普通数组区别

- 不同之处在于数组是静态空间，而vector可以动态扩展。

- 当空间不足时，vector会自动重新申请更大的内存，将旧空间数据拷贝到新空间，然后自动释放内存。

- vector<T> v(v1.begin(), v1,end()) 将v1[begin(), end())区间中的元素拷贝给v。不包含end()，因为end()在最后一个元素之后。
- vector<T> v(n, element) 构造v时，将n个element拷贝给v
- vector<T> v(const vector& v1) 拷贝构造函数

- vector& operator=(const vector& v)

## 容量和大小

- 容量指在不分配新内存空间的情况下，可以存储元素的最大个数。
- 大小指当前已经使用存储空间数量。
- 为了减少vector在动态扩容时的扩展次数，可以指定扩容的大小。
- empty()判断是否为空
- capacity()获取容器容量
- size()获取容器大小
- resize()指定容器的长度

- reserve()预留一定的长度

## 插入和删除

- push_back(element)尾部插入元素elment

- insert(pos, element)指定位置插入1个元素element

## 数据存取

- at(int idx)返回索引idx指定的数据
- [idx]返回索引idx指定的数据

- back()返回容器中最后一个数据

## 互换容器

- swap(v2) 将参数容器v2与自身元素互换

## vector的特点

- vector在尾部插入和删除效率高，所以经常被称为单端容器
- 与其他动态序列容器相比，vector在访问元素时更加高效，但是在指定位置插入时可能引起大量数据移动，效率低。
- 与数组相似，vector也采用连续内存空间存储元素，可以采用下标对元素进行访问；与数组不同的是它的大小可以改变。

## 常用算法函数

- sort(begin, end)对begin和end区间元素进行排序
- sort(begin, end, func)对begin和end区间元素进行排序,用func函数指定排序规则
- reverse(begin, end)对begin和end区间元素进行反转倒置
- find(begin, end, element)在begin和end区间，查找元素element
- copy(源起始位置，源的结束位置，目标的起始位置)选取源的一段内容复制到目标内容，例如：copy(v1.begin，v1.end，v2.begin())

## 遍历vector

- 可以使用迭代器，for_each算法和下标遍历vector中的元素

- 迭代器(也叫位置指针): vector::iterator

  - 迭代器允许在不暴露集合底层表现的情况下，顺序访问一个集合中的元素，主要优点为：

    - 简化集合遍历

    - 隐藏集合内部实现

    - 简化客户端代码

    - 支持延迟加载

# 十一、容器使用时机

- 选择合适的STL容器应综合考虑数据访问模式，插入、删除频率，内存布局要求以及性能特性，并结合具体业务场景进行权衡。

## vector

- 适用于元素数量动态增长，主要在尾部进行插入、访问且很少在中间或头部修改的场景。

- 应用场景：存储日志，历史操作记录，这类数据通常只需要追加，几乎不删除或随机修改。

## deque

- 支持在首尾端高效地插入和删除，但不支持高效的中间操作。

- 应用场景：用于维护最近N条记录，例如游戏中的技能冷却，多个技能出发后，按触发顺序依次生效。

## stack

- 先进后出，数据只能从最上面访问。

- 应用场景：撤销功能，输入就压栈，撤销就弹出最近的操作。

## queue

- 先进先出，数据拍到队尾，从队头出队

- 应用场景：排队系统，新请求从尾部加入，处理完成请求从头部移除。

## list

- 适合频繁在任意位置插入或删除元素的场景。

- 应用场景：模拟公交车乘客上下车，增删改频繁，操作位置也不缺定。

## set

- 基于红黑树实现，支持高效的插入、删除和查找操作。 
- 应用场景：游戏积分榜，频繁变化。

## map

- 基于红黑树实现，支持根据键进行高效的插入、删除和查找操作。 

- 应用场景：按用户id查找用户信息。

  - 谓词是一种特殊的函数对象，它的返回值是bool类型。

  - 谓词还可以进一步分为，一元谓词(一个参数)和二元谓词(两个参数)

  - 谓词的参数常为只读

- 函数对象本上是一个特殊类的对象，它内部重载了()运算符，因此和访问函数相似，也被称为仿函数。

- 函数对象可以当作参数传递，常用于指定容器的排序规定

- 函数对象可以有返回值、参数也可以有自己的内部变量。

- STL内建了一些常用的函数对象

  - 算术仿函数

    - T plus<T> 加法仿函数

    - T minus<T> 减法仿函数

    - T multiplies<T> 乘法仿函数

    - T divides<T> 除法仿函数

    - T negate<T> 取反仿函数

    - T modulus<T> 取模仿函数，返回第一个参数除以第二个参数的余数。

  - 关系仿函数

    - bool equal_to<T> 等于

    - bool not_equal_to<T> 不等于

    - bool greater<T> 大于

    - bool greater_equal<T> 大于等于

    - bool less<T> 小于

    - bool less_equal<T> 不等于

  - 逻辑仿函数

    - bool logical_and<T> 逻辑与

    - bool logical_not<T> 逻辑非

    - bool logical_or<T> 逻辑或
