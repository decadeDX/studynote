# AI 操作命令语言指南

> 一份面向嵌入式/软件开发者的 AI 提问方法论，帮你用最短时间获得最精准的答案。

---

# 一、核心原则：AI 不是搜索引擎

| 思维误区 | 正确认知 |
|---------|---------|
| AI 什么都知道，随便问就行 | AI 需要**上下文**才能给出精准答案 |
| 问得越短越快越好 | 问得越**具体**，回答越有用 |
| AI 给的代码直接复制用 | AI 是**辅助工具**，你需要理解后再用 |

**一句话**：把 AI 当 **senior 同事**，而不是魔法盒子。你说得越清楚，它帮得越到位。

# 二、四大黄金提问模板

## 2.1 模板一：解释概念（What / Why）

适用场景：学习新知识，理解某个概念、原理。

```
【指令格式】
解释 [概念/术语]，要求：
1. 用一句话总结是什么
2. 用类比/图示说明核心原理
3. 举一个实际应用的例子
4. 和我正在学的 [相关知识] 有什么关联
```

**示例**（结合你的学习内容）：

> 解释 FreeRTOS 中的任务调度器，要求：
> 1. 用一句话总结它是什么
> 2. 用生活中排队的类比说明优先级抢占
> 3. 给一个 STM32 上创建两个不同优先级任务的代码例子
> 4. 和裸机程序的 while(1) 循环有什么区别

---

## 2.2 模板二：生成代码（How to implement）

适用场景：需要写某个功能、某个驱动、某个算法。

```
【指令格式】
用 [语言] 实现 [功能描述]，要求：
1. 运行环境：[芯片/系统/编译器]
2. 输入：[参数类型和含义]
3. 输出：[返回值/效果]
4. 约束：[性能要求/内存限制/编码规范]
5. 给每个关键步骤加注释
```

**示例**：

> 用 C 语言实现一个 GPIO 按键消抖函数，要求：
> 1. 运行环境：STM32F103，HAL 库
> 2. 输入：GPIO_Pin 引脚号
> 3. 输出：按键按下返回 1，未按下返回 0
> 4. 消抖时间 20ms，用非阻塞方式实现（不能死等 delay）
> 5. 给关键步骤加中文注释

---

## 2.3 模板三：调试/排错（Debug）

适用场景：代码出 bug、编译报错、运行异常。

```
【指令格式】
我遇到了一个问题：[描述现象]
1. 错误信息/日志：[贴完整报错]
2. 相关代码：[贴关键代码片段]
3. 运行环境：[芯片/系统/版本]
4. 我已尝试：[你排查过的方向]
请帮我：先分析原因，再给出修复方案
```

**示例**：

> 我遇到了一个问题：STM32 串口接收数据时偶尔丢字节。
> 1. 错误现象：每收到 100 字节大概丢 2-3 个，没有报错
> 2. 相关代码：
> ```c
> void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart) {
>     HAL_UART_Receive_IT(&huart1, &rx_byte, 1);
>     ring_buf_write(&buf, rx_byte);
> }
> ```
> 3. 运行环境：STM32F103C8T6，波特率 115200，HAL 库
> 4. 我已尝试：降低波特率到 9600 后正常，但业务需要 115200
> 请帮我：先分析可能原因，再给出修复方案

## 2.4 模板四：总结/梳理知识（Organize）

适用场景：学完一个模块后，让 AI 帮你整理笔记、画知识图谱。

```
【指令格式】
帮我整理 [知识点] 的笔记，要求：
1. 列出核心概念清单（层级关系）
2. 用表格对比容易混淆的概念
3. 画一个 Mermaid 流程图展示调用/执行流程
4. 补充常见面试题/易错点
```

**示例**：

> 帮我整理 C 语言指针的笔记，我从 [[嵌入式学习笔记]] 中学了以下内容：
> - 指针定义、多级指针、指针运算
> - 常量指针 vs 指针常量
> - 函数指针、回调函数
> - 动态内存分配(malloc/free)
> 要求：
> 1. 用表格对比常量指针和指针常量（语法、特点、使用场景）
> 2. 画一个 Mermaid 图展示 malloc -> 使用 -> free 的完整流程和注意事项
> 3. 补充常见的指针相关面试题

## 3.4 Java 开发

### 集合选型
```
我需要存储一组数据，场景如下：
- 数据特点：[有序/无序、可重复/不可重复、键值对/单列]
- 主要操作：[频繁查询 / 频繁增删 / 需要排序 / 需要线程安全]
- 数据量级：[预估元素个数]
帮我选择合适的集合类，并用表格对比候选方案（底层结构、时间复杂度、线程安全）
```

**示例**：
> 我需要存储10万个用户对象，场景如下：
> - 数据特点：单列、有序、可重复
> - 主要操作：按索引频繁随机访问（占90%），偶尔尾部追加
> - 数据量级：10w
> 帮我选择合适的 List 实现，并说明为什么不用 LinkedList

### 线程并发调试
```
我遇到了一个多线程问题：[描述现象（死锁/数据不一致/性能瓶颈）]
1. 相关代码：[贴关键代码片段]
2. 线程数/线程池配置：[corePoolSize, maxPoolSize, 队列类型]
3. 现象：[偶发死锁 / 结果每次不同 / CPU 100% / 吞吐量上不去]
帮我从 [可见性/原子性/有序性/死锁四条件/线程池参数] 角度分析根因
```

**示例**：
> 我遇到了一个多线程问题：两个线程转账偶尔出现总金额不对。
> 1. 相关代码：
> ```java
> public void transfer(Account from, Account to, int amount) {
>     from.balance -= amount;
>     to.balance += amount;
> }
> ```
> 2. 线程数：10 个线程同时转账
> 3. 现象：总金额偶尔变多或变少，不是每次都错
> 帮我从原子性和可见性角度分析根因，并给出 synchronized 和 Lock 两种修复方案

### Spring / Controller 开发
```
帮我写一个 Spring Boot 的 Controller：
- 接口路径：[POST /api/v1/user]
- 接收参数：[@RequestHeader token, @RequestBody UserDto, @PathVariable id]
- 返回格式：[统一返回 Result<T>，包含 code, msg, data]
- 需要处理的异常：[参数校验失败 / 业务异常 / 系统异常]
- 其他要求：[参数校验用 @Valid / 日志用 @Slf4j]
```

### String / 日期处理
```
帮我优化下面这段 Java 字符串处理代码：
[贴代码]
要求：
1. 如果在循环中拼接字符串，改用 StringBuilder
2. 如果用 SimpleDateFormat，改为 DateTimeFormatter（线程安全）
3. 如果有 null 风险，加上 Objects.requireNonNull 或 Optional
```

### 异常处理审查
```
帮我审查这段代码的异常处理是否合理：
[贴代码]
从以下角度分析：
1. 有没有吞异常（catch 后什么都不做）
2. 有没有在 finally 里写 return（会吞掉异常）
3. try-with-resources 是否可以用（简化资源关闭）
4. 自定义异常是否继承对了类型（Exception vs RuntimeException）
```

# 四、常用速查指令

## 4.1 一句搞定（适合快速提问）

|     | 场景               | 指令                                                           |
| --- | ---------------- | ------------------------------------------------------------ |
|     | 查语法              | `[语言] 中 [语法点] 的正确写法是什么？给一个例子`                                |
|     | 翻译报错             | `这个报错是什么意思：[贴报错信息]`                                          |
|     | 找替代方案            | `[当前做法] 有什么缺点？有没有更好的替代方案？`                                   |
|     | 要示例代码            | `给我一个 [语言] 的 [功能] 最小可运行示例`                                   |
|     | 求解释差异            | `[A] 和 [B] 有什么区别？用表格对比`                                      |
|     | 画流程图             | `用 Mermaid 画一个 [流程名] 的流程图`                                   |
|     | 写 Makefile/CMake | `帮我写一个 [场景] 的 CMakeLists.txt`                                |
|     | 寄存器配置            | `[芯片] 的 [外设] 寄存器怎么配置才能实现 [功能]`                               |
|     | 集合选型             | `[场景] 下用 ArrayList 还是 LinkedList？用 HashMap 还是 TreeMap？用表格对比` |
|     | 线程问题             | `这段多线程代码有什么问题？从可见性/原子性/有序性三个角度分析`                            |
|     | 死锁排查             | `帮我分析这两个线程会不会死锁，画一个资源等待图`                                    |
|     | 线程池调优            | `我的任务是 [CPU密集/IO密集]，QPS 约 [X]，帮我计算线程池参数`                     |
|     | String优化         | `这段字符串拼接代码有什么性能问题？帮我用 StringBuilder 优化`                      |
|     | 日期处理             | `Java 中 [日期需求] 用 Date 还是 LocalDateTime？给出线程安全的写法`            |
|     | Lombok           | `帮我用 Lombok 简化这个类：[贴类代码]，要求用 @Data @Builder @Slf4j`          |

## 4.2 约束词（加在指令末尾，控制输出）

```
用通俗易懂的语言解释          → 降低难度
用专家级别的语言回答           → 提高深度
只给关键思路，不需要完整代码   → 精简输出
给出完整可运行的代码           → 完整输出
用表格形式组织答案             → 结构化
分步骤说明                     → 流程化
每种方案给出优缺点             → 对比决策
加上 Mermaid 时序图展示调用链  → 可视化交互流程
画出线程状态转换图             → 可视化并发逻辑
用 jstack/jmap 分析思路排查    → 生产环境排障
```

# 五、避坑指南

| 坑 | 正确做法 |
|----|---------|
| ❌ 一次性问太多不相关的问题 | ✅ 一个问题只聚焦一个主题 |
| ❌ 不说运行环境让 AI 猜 | ✅ 先说芯片型号、编译器、系统版本 |
| ❌ 贴几百行代码让 AI 找 bug | ✅ 只贴出错相关的关键代码 |
| ❌ AI 给的代码不经理解直接跑 | ✅ 先读懂逻辑，再适配你的工程 |
| ❌ 用模糊词"那个东西""这个" | ✅ 用准确的术语和变量名 |
| ❌ 在同一个对话里切换完全不同的主题 | ✅ 开新对话问新主题 |
| ❌ 线程问题只描述"偶尔出bug" | ✅ 说明频率、触发条件、是否可复现、线程数 |
| ❌ 让 AI 直接选集合类不说场景 | ✅ 先说数据特点、主要操作、数据量级 |
| ❌ 字符串拼接性能问题不贴循环代码 | ✅ 贴出循环体，说明循环次数级别（百/万/百万） |
| ❌ Spring 报错只贴最后一行 | ✅ 贴 Caused by 链 + 相关 Bean 的注入方式 |

# 六、结合 Obsidian 的工作流

## 6.1 学习新知识时

```
1. 用 AI 提问获得初步理解
2. 将 AI 的回答整理成 Obsidian 笔记（用你自己的话写）
3. 用 [[双向链接]] 关联到已有笔记
4. 用表格/Mermaid 图可视化关键逻辑
```

## 6.2 写代码时

```
1. 用 AI 生成代码框架/模板
2. 在工程中调试、修改
3. 把踩坑经验写回 Obsidian 笔记
4. 记录：[问题现象] → [排查过程] → [最终方案]
```

## 6.3 复习时

```
1. 给 AI 贴上你的笔记内容
2. 让 AI 根据笔记出几道面试题 / 测试题
3. 自己回答后让 AI 点评
```

# 七、终极口诀

> **角色 + 场景 + 约束 = 好答案**

- **角色**：你在做什么（学 Linux 驱动 / 写 STM32 代码 / 准备面试）
- **场景**：具体是什么问题（中断丢失 / 编译报错 / 不理解某个概念）
- **约束**：你需要什么形式（要代码 / 要解释 / 要对比 / 要流程图）
