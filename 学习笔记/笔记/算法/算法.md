# 查找范围内的素数

## 算法

埃式筛法

1.把2到n范围内的整数写下来

2.其中2是最小的素数。故将表中所有的2的倍数划去

3.表中剩下的最小的数字就是3。再划去3的倍数

4.以此类推，如果表中剩余的最小的数是m，那么m就是素数。然后将表中所有m的倍数划去，像这样反复操作，就能依次枚举n以内的素数

埃氏筛法：时间复杂度是O(nloglogn)。

```c
#include <stdio.h>
#include <stdlib.h>

#define MAX 1000

int main()
{
    int prime[MAX] = {0};//素数标记数组
    for (int i = 2; i < MAX; i++)//从2开始，0、1不是素数
    {
        prime[i] = 1;//默认都是素数,1表示素数、0表示不是素数
    }
    for (int i = 2; i * i < MAX ; i++)//设我们要判断一个数 n是不是合数，核心结论是：如果 n 是合数，
        							  //那它一定有一个因数小于等于（开根号n）。
          							  //反过来说就是：如果一个数n在 2~（开根号n）范围内没有任何因数，那它一定是素数。
    {
        if(prime[i])
        {
            /*
            为什么内层循环从 i×i开始，而不是从 2×i开始？因为2×i 、 3×i、...、(i−1)×i这些 i 的倍数，
            早已被比 i 更小的素数标记过了，
            从i×i 开始能避免重复标记，进一步减少循环次数。
            
            每次加i：
            目的是依次标记素数i的所有整数倍。因为素数i的倍数就是i×i、i×(i+1)、i×(i+2)、i×(i+3)...，
            这些数之间的差值都是i，
            所以每次让j加i，就能精准遍历到i的所有倍数，不会遗漏、也不会多余。
            */
            for (int j = i * i; j < MAX; j += i)
            {
                prime[j] = 0;
            }
        }
    }
    //下标对应相应的数，如果prime[i] == 1,则i是素数。
}

//时间复杂度O（nloglogn）
```
